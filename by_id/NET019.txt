DECK: Networking::linux_namespaces
QUESTION: COMPARE: How would you explain namespaces vs containers vs VMs to someone new?
ANSWER: <b>ISOLATION SPECTRUM (least to most):</b><br><br><b>PROCESSES (no isolation):</b><br>• Share everything: network, filesystem, memory<br>• One crash can affect others<br><br><b>NETWORK NAMESPACES (network isolation only):</b><br>• Isolated: Network stack<br>• Shared: Filesystem, CPU, memory<br><br><b>CONTAINERS (namespace bundle):</b><br>• Isolated: Network + PID + filesystem + users<br>• Shared: Kernel, CPU resources (cgroups limit)<br><br><b>VIRTUAL MACHINES (full isolation):</b><br>• Isolated: EVERYTHING including kernel<br>• Own OS, own kernel, own memory<br><br><b>KEY INSIGHT:</b><br>• Namespace = one isolation dimension<br>• Container = multiple namespaces bundled<br>• VM = complete isolation but heavier<br><br><b>RULE OF THUMB:</b><br>• Same kernel? → Namespaces possible<br>• Different kernel? → Need VM<br>• Docker on Windows? → Actually runs in Linux VM (WSL2)
