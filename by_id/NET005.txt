QUESTION: WHAT is a routing table and HOW does the kernel use it to decide where to send packets?
ANSWER: <b>DEFINITION:</b> A set of rules telling the kernel "to reach destination X, send via interface Y through gateway Z."<br><br><b>THE DECISION PROCESS:</b><br>1. Kernel receives outgoing packet for IP X<br>2. Scans routing table top-to-bottom<br>3. Finds FIRST matching rule (most specific wins)<br>4. Sends packet via that interface/gateway<br><br><b>EXAMPLE TABLE:</b><br>192.168.1.0/24 → wlan0 (local network)<br>10.0.0.0/8 → eth0 (VPN network)<br>0.0.0.0/0 → gateway 192.168.1.1 (default route)<br><br><b>SPECIAL ENTRIES:</b><br>• 0.0.0.0/0 = Default route (catch-all)<br>• Gateway 0.0.0.0 = "directly connected"<br>• Gateway with IP = "send to this router first"<br><br><b>KEY INSIGHT:</b> More specific routes (/24) take precedence over less specific (/0). This is how VPNs work - they add more specific routes.
